Network Fundamentals

Baseline, interface, IPv4/IPv6, dan troubleshooting fisik.

20%
01
Baseline perangkatIdentity · password · banner · save
enable
configure terminal
hostname <HOSTNAME>
no ip domain-lookup
enable secret <ENABLE_SECRET>
service password-encryption
username <USER> privilege 15 secret <USER_SECRET>
banner motd #AKSES HANYA UNTUK PENGGUNA RESMI#
end
copy running-config startup-config

service password-encryption hanya mengobfuscate password plaintext; gunakan secret untuk credential lokal.

02
IPv4 & IPv6 interfaceAddress · EUI-64 · link-local
configure terminal
ipv6 unicast-routing
interface <INTERFACE>
 description <DESCRIPTION>
 ip address <IPV4> <MASK>
 ipv6 address <IPV6>/<PREFIX>
 ipv6 address <PREFIX64>/64 eui-64
 ipv6 address fe80::1 link-local
 no shutdown
end
show ip interface brief
show ipv6 interface brief
show interfaces <INTERFACE>

Pilih salah satu metode global IPv6 manual atau EUI-64; jangan memasang dua global address tanpa alasan desain.

03
Interface troubleshootingStatus · duplex · CRC · counters

Verifikasi awal:

show interfaces status
show interfaces <INTERFACE>
show interfaces counters errors
show controllers <INTERFACE>

Pulihkan speed/duplex otomatis bila desain memang memakai autonegotiation:

configure terminal
interface <INTERFACE>
 speed auto
 duplex auto
 no shutdown
end
show interfaces <INTERFACE>

Command show interfaces status dan show controllers tersedia pada platform tertentu. CRC/input errors mengarah ke media/transceiver atau mismatch; late collision sering menunjukkan duplex mismatch atau segment terlalu panjang.

04
Switching & MAC tableLearning · aging · flooding
show mac address-table
show mac address-table dynamic
show mac address-table interface <INTERFACE>
show interfaces <INTERFACE> switchport
show mac address-table aging-time

Switch mempelajari source MAC pada port masuk. Unknown unicast dan broadcast di-flood hanya dalam VLAN yang sama. clear mac address-table dynamic sengaja tidak dimasukkan ke blok salin karena menghapus entry dinamis dan dapat memicu flooding sementara; jalankan hanya saat troubleshooting terkontrol.

VLAN, trunk, inter-VLAN, discovery, EtherChannel, dan STP.

20%
05
VLAN, access & voiceNormal-range VLAN · port membership
configure terminal
vlan <DATA_VLAN>
 name <DATA_NAME>
vlan <VOICE_VLAN>
 name <VOICE_NAME>
interface <ACCESS_PORT>
 switchport mode access
 switchport access vlan <DATA_VLAN>
 switchport voice vlan <VOICE_VLAN>
 spanning-tree portfast
end
show vlan brief

06
802.1Q trunkNative VLAN · allowed list
configure terminal
interface <TRUNK_PORT>
 switchport mode trunk
 switchport trunk native vlan <NATIVE_VLAN>
 switchport trunk allowed vlan <LIST>
 no shutdown
end
show interfaces trunk
show interfaces <TRUNK_PORT> switchport

Perhatian: switchport trunk allowed vlan <LIST> mengganti daftar yang ada. Untuk menambahkan VLAN tanpa menimpa daftar, gunakan switchport trunk allowed vlan add <VLAN>. Native dan allowed VLAN harus konsisten pada kedua ujung. Pada platform lama mungkin diperlukan switchport trunk encapsulation dot1q.

07
Inter-VLAN routingRouter-on-a-stick · SVI · routed port

Opsi A — router-on-a-stick:

configure terminal
interface <PHYSICAL>
 no shutdown
interface <PHYSICAL>.<VLAN_ID>
 encapsulation dot1q <VLAN_ID>
 ip address <GATEWAY> <MASK>
end
show ip interface brief

Opsi B — SVI pada multilayer switch:

configure terminal
vlan <VLAN_ID>
ip routing
interface vlan <VLAN_ID>
 ip address <GATEWAY> <MASK>
 no shutdown
end
show ip interface brief

Opsi C — routed port:

configure terminal
ip routing
interface <L3_PORT>
 no switchport
 ip address <IPV4> <MASK>
 no shutdown
end
show ip interface brief

Pilih satu opsi sesuai topologi. SVI memerlukan VLAN aktif dan minimal satu member/trunk yang forwarding agar line protocol up. Dukungan no switchport dan ip routing bergantung model/image switch.

08
CDP & LLDPLayer 2 neighbor discovery
configure terminal
cdp run
lldp run
interface <INTERFACE>
 cdp enable
 lldp transmit
 lldp receive
end
show cdp neighbors
show cdp neighbors detail
show lldp neighbors
show lldp neighbors detail

Untuk menonaktifkan CDP hanya pada satu interface gunakan no cdp enable; no cdp run menonaktifkannya secara global. Dukungan detail LLDP/CDP dapat berbeda menurut platform.

09
LACP EtherChannelLayer 2 dan Layer 3 bundle

Opsi A — Layer 2 trunk EtherChannel:

configure terminal
interface range <MEMBER_RANGE>
 switchport mode trunk
 channel-group <GROUP> mode active
interface port-channel <GROUP>
 switchport mode trunk
 no shutdown
end
show etherchannel summary
show lacp neighbor

Opsi B — Layer 3 EtherChannel:

configure terminal
ip routing
interface range <MEMBER_RANGE>
 no switchport
 channel-group <GROUP> mode active
interface port-channel <GROUP>
 no switchport
 ip address <IPV4> <MASK>
 no shutdown
end
show etherchannel summary
show lacp neighbor
show etherchannel load-balance

Pilih satu opsi dan terapkan konfigurasi yang kompatibel pada kedua ujung. active + passive atau active + active membentuk LACP; passive + passive tidak. Parameter seluruh member harus konsisten.

10
Rapid PVST+ & guardsRoot · roles · PortFast · protection

SW1 — kandidat root utama:

configure terminal
spanning-tree mode rapid-pvst
spanning-tree vlan <VLAN> root primary
end
show spanning-tree root

SW2 — kandidat root cadangan:

configure terminal
spanning-tree mode rapid-pvst
spanning-tree vlan <VLAN> root secondary
end
show spanning-tree root

Proteksi port — pilih sesuai peran port:

configure terminal
interface <HOST_PORT>
 spanning-tree portfast
 spanning-tree bpduguard enable
interface <DESIGNATED_PORT>
 spanning-tree guard root
interface <NON_DESIGNATED_PORT>
 spanning-tree guard loop
end
show spanning-tree vlan <VLAN>
show spanning-tree interface <PORT> detail

BPDU Filter dapat menyembunyikan BPDU dan menimbulkan loop; gunakan hanya bila konsekuensinya dipahami.

Checklist Wireless GUI: buat WLAN/SSID → map ke VLAN/interface → pilih WPA2-PSK/AES atau WPA3-SAE sesuai dukungan → atur QoS/profile → enable WLAN → verifikasi association, authentication, DHCP, gateway, DNS, dan roaming. Menu bergantung model WLC; praktikkan pada Lab 38, Lab 44, dan Lab 57.

IP Connectivity

Routing table, static route IPv4/IPv6, dan single-area OSPFv2.

25%
11
IPv4 static routeNetwork · default · host · floating

Pilih hanya jenis route yang diperlukan:

configure terminal
! Network route
ip route <NETWORK> <MASK> <NEXT_HOP>
! Default route
ip route 0.0.0.0 0.0.0.0 <NEXT_HOP>
! Host route
ip route <HOST_IP> 255.255.255.255 <NEXT_HOP>
! Floating static
ip route <NETWORK> <MASK> <BACKUP_NEXT_HOP> 200
end
show ip route
show ip route static
show ip route <DESTINATION>
show ip cef <DESTINATION>
12
IPv6 static routeDefault · host · link-local next hop

Aktifkan forwarding lalu pilih hanya jenis route yang diperlukan:

configure terminal
ipv6 unicast-routing
ipv6 route <PREFIX>/<LENGTH> <NEXT_HOP>
ipv6 route ::/0 <NEXT_HOP>
ipv6 route <HOST>/128 <NEXT_HOP>
ipv6 route <PREFIX>/<LENGTH> <INTERFACE> <LINK_LOCAL>
ipv6 route <PREFIX>/<LENGTH> <BACKUP_NEXT_HOP> 200
end
show ipv6 route
show ipv6 route static
show ipv6 route <DESTINATION>
show ipv6 cef <DESTINATION>

Jika next hop adalah link-local, exit interface wajib disebut.

13
Single-area OSPFv2RID · passive · point-to-point · DR/BDR

Konfigurasi dasar + interface point-to-point:

configure terminal
router ospf <PROCESS_ID>
 router-id <ROUTER_ID>
 passive-interface default
 no passive-interface <TRANSIT_INTERFACE>
 default-information originate

interface <INTERFACE>
 ip ospf <PROCESS_ID> area 0
 ip ospf network point-to-point
 ip ospf cost <COST>
end
show ip ospf neighbor
show ip ospf interface brief
show ip route ospf

Khusus broadcast multiaccess, atur eligibility DR/BDR bila diperlukan:

configure terminal
interface <BROADCAST_INTERFACE>
 no ip ospf network point-to-point
 ip ospf priority <0-255>
end
show ip ospf interface <BROADCAST_INTERFACE>
show ip ospf neighbor
show ip ospf database
show ip route ospf

ip ospf priority memengaruhi pemilihan DR/BDR pada network type yang melakukan election, bukan point-to-point. default-information originate memerlukan default route di RIB kecuali memakai always. Jika router ID diubah setelah proses aktif, restart proses OSPF pada maintenance window atau reload agar identitas baru digunakan.

14
Routing decisionRIB selection · LPM · FHRP verify
show ip route <DESTINATION>
show ipv6 route <DESTINATION>
show ip protocols
show ip cef <DESTINATION> detail
traceroute <DESTINATION>
traceroute ipv6 <DESTINATION>
show standby brief

RIB: untuk prefix identik, administrative distance memilih sumber route dan metric memilih jalur dalam protocol yang sama. Forwarding: destination paket dicocokkan ke entry RIB dengan longest prefix match. show standby brief hanya untuk HSRP dan ketersediaannya bergantung platform; blueprint menekankan konsep FHRP.

15
Static NAT, pool & PATTiga skenario inside source

Opsi A — static NAT one-to-one:

configure terminal
interface <LAN_INTERFACE>
 ip nat inside
interface <WAN_INTERFACE>
 ip nat outside
ip nat inside source static <INSIDE_LOCAL> <INSIDE_GLOBAL>
end
show ip nat translations
show ip nat statistics

Opsi B — dynamic NAT dengan pool:

configure terminal
interface <LAN_INTERFACE>
 ip nat inside
interface <WAN_INTERFACE>
 ip nat outside
access-list 1 permit <INSIDE_NETWORK> <WILDCARD>
ip nat pool <POOL> <START_GLOBAL> <END_GLOBAL> netmask <MASK>
ip nat inside source list 1 pool <POOL>
end
show ip nat translations
show ip nat statistics

Opsi C — PAT memakai alamat interface WAN:

configure terminal
interface <LAN_INTERFACE>
 ip nat inside
interface <WAN_INTERFACE>
 ip nat outside
access-list 1 permit <INSIDE_NETWORK> <WILDCARD>
ip nat inside source list 1 interface <WAN_INTERFACE> overload
end
show ip nat translations
show ip nat statistics

Pilih satu skenario sesuai desain dan pastikan routing dua arah tersedia. clear ip nat translation * tidak disertakan pada blok salin karena memutus translation aktif; gunakan hanya saat troubleshooting terkontrol.

16
NTP client & serverTime synchronization

Perangkat sebagai NTP client:

configure terminal
clock timezone WIB 7 0
service timestamps log datetime msec
ntp server <NTP_SERVER> prefer
ntp server <BACKUP_NTP>
end
show clock detail
show ntp associations
show ntp status

Hanya untuk lab terisolasi sebagai authoritative source:

configure terminal
ntp master <STRATUM>
end
show ntp status

ntp master membuat perangkat melayani waktu dari clock lokal; jangan gunakan sebagai pengganti sumber waktu tepercaya pada produksi. Sintaks timezone dan NTP dapat berbeda menurut platform.

17
DHCP client & relayDynamic address · helper

Opsi A — interface IOS sebagai DHCP client:

configure terminal
interface <WAN_INTERFACE>
 ip address dhcp
 no shutdown
end
show dhcp lease
show ip interface brief

Opsi B — relay pada interface gateway client:

configure terminal
interface <CLIENT_GATEWAY_INTERFACE>
 ip helper-address <DHCP_SERVER>
end
show ip interface brief
show running-config interface <INTERFACE>

ip helper-address diletakkan pada interface yang menerima broadcast client; server harus memiliki pool dan return route untuk subnet giaddr.

18
SSH remote accessLocal user · RSA · VTY
configure terminal
hostname <HOSTNAME>
ip domain-name <DOMAIN>
username <USER> privilege 15 secret <SECRET>
crypto key generate rsa modulus 2048
ip ssh version 2
line vty 0 15
 login local
 transport input ssh
 exec-timeout 10 0
end
show ip ssh
show ssh

Jumlah line VTY dan dukungan modulus RSA dapat berbeda menurut image/platform. Gunakan secret unik dan jangan menempel credential produksi dari halaman web.

19
Syslog & SNMPPolling · traps · event logging
configure terminal
service timestamps log datetime msec localtime
logging host <SYSLOG_SERVER>
logging trap warnings
logging buffered 16384 notifications

ip access-list standard NMS-ONLY
 permit host <NMS_IP>
snmp-server community <COMMUNITY> ro NMS-ONLY
snmp-server host <NMS_IP> version 2c <COMMUNITY>
snmp-server enable traps
end
show logging
show snmp community
show snmp host

NMS melakukan polling terhadap agent; snmp-server host menetapkan penerima notification, sedangkan snmp-server enable traps mengaktifkan kategori trap yang didukung. Sintaks trap bervariasi menurut image. SNMPv2c community adalah plaintext; prioritaskan SNMPv3 pada lingkungan nyata.

20
TFTP/FTP backupSave · export · restore

Simpan lokal dan ekspor backup:

copy running-config startup-config
copy running-config tftp:
copy startup-config tftp:
copy flash: ftp:
dir flash:
show file systems

Restore — jalankan satu arah copy setelah backup dan verifikasi file:

copy tftp: running-config
copy tftp: startup-config

Perhatian: copy ke running-config melakukan merge langsung dan dapat mengubah akses/routing saat itu juga; copy ke startup-config memengaruhi boot berikutnya. Verifikasi source, destination, checksum/isi, dan siapkan rollback. TFTP tidak memiliki authentication/encryption; FTP memakai authentication tetapi credential/data tidak terenkripsi secara default.

QoS PHB: gunakan show policy-map interface untuk verifikasi. Konfigurasi class-map/policy-map bersifat pengayaan; scope ujian menekankan classification, marking, queuing, congestion, policing, dan shaping.

Security Fundamentals

Device access, ACL, port security, DHCP snooping, dan DAI.

15%
21
Local access controlSecret · console · VTY · ACL
configure terminal
enable secret <ENABLE_SECRET>
username <ADMIN> privilege 15 secret <ADMIN_SECRET>
line console 0
 login local
 exec-timeout 5 0
line vty 0 15
 login local
 transport input ssh

ip access-list standard MGMT
 permit <MGMT_NETWORK> <WILDCARD>
 deny any log
line vty 0 15
 access-class MGMT in
end
show access-lists MGMT
show running-config | section line vty

ACL management harus mengizinkan alamat admin yang benar sebelum diterapkan agar sesi berikutnya tidak terkunci. SSH juga memerlukan hostname, domain name, dan RSA key seperti pada kartu 18.

22
IPv4 & IPv6 ACLStandard · extended · traffic-filter

Opsi A — standard IPv4 ACL:

configure terminal
ip access-list standard <STD_ACL>
 permit <SOURCE> <WILDCARD>
 deny any log
interface <INTERFACE>
 ip access-group <STD_ACL> in
end
show access-lists <STD_ACL>

Opsi B — extended IPv4 ACL untuk policy HTTPS tertentu:

configure terminal
ip access-list extended <EXT_ACL>
 permit tcp <SOURCE> <WC> host <SERVER> eq 443
 deny ip <SOURCE> <WC> any log
 permit ip any any
interface <INTERFACE>
 ip access-group <EXT_ACL> in
end
show access-lists <EXT_ACL>

Opsi C — contoh IPv6 ACL inbound pada VLAN client:

configure terminal
ipv6 access-list <IPV6_ACL>
 permit icmp any any router-solicitation
 permit icmp any any nd-ns
 permit icmp any any nd-na
 permit icmp any any packet-too-big
 permit icmp any any time-exceeded
 permit icmp any any parameter-problem
 permit tcp <SOURCE_PREFIX>/64 host <SERVER_V6> eq 443
 deny ipv6 any any log
interface <INTERFACE>
 ipv6 traffic-filter <IPV6_ACL> in
end
show ipv6 access-list
show ipv6 interface <INTERFACE>

Setiap blok adalah contoh policy terpisah; sesuaikan interface, arah, source, destination, dan kebutuhan trafik. Contoh IPv6 mengizinkan Router Solicitation dari client tetapi sengaja tidak mengizinkan Router Advertisement masuk dari client. Jangan memblokir Neighbor Discovery atau ICMPv6 yang diperlukan Path MTU Discovery. Semua ACL memiliki implicit deny di akhir.

23
Port securityMaximum · sticky · violation
configure terminal
interface <HOST_PORT>
 switchport mode access
 switchport access vlan <VLAN>
 switchport port-security
 switchport port-security maximum <COUNT>
 switchport port-security mac-address sticky
 switchport port-security violation restrict
end
show port-security
show port-security interface <HOST_PORT>
show port-security address

Sticky MAC tidak digunakan untuk latihan dynamic aging. Violation mode: protect, restrict, atau shutdown.

24
DHCP snooping & DAITrusted uplink · rate limiting
configure terminal
ip dhcp snooping
ip dhcp snooping vlan <VLAN_LIST>
ip arp inspection vlan <VLAN_LIST>

interface <TRUSTED_UPLINK>
 ip dhcp snooping trust
 ip arp inspection trust

interface range <HOST_PORTS>

end

DAI menggunakan DHCP snooping binding. Host statis memerlukan ARP ACL atau mekanisme binding yang sesuai.

Security tanpa satu IOS template: threat/vulnerability/exploit/mitigation, awareness, physical access, MFA/certificate/biometric, IPsec remote/site-to-site, dan WPA2/WPA3 dinilai sebagai konsep/desain atau melalui GUI. Lihat Lab 18, Lab 37, Lab 44, Lab 53, dan Lab 55.

Command ini mengumpulkan state yang dapat dibandingkan oleh automation. Filter pipe, show inventory, dan terminal length dapat berbeda menurut platform. Jangan menganggap output CLI sebagai JSON tanpa parser atau API yang mendukung structured data.

26
API & automation boundaryREST · JSON/XML · Ansible · Terraform

Bukan Cisco IOS CLI langsung:

TujuanMekanisme
Create / Read / Update / DeletePOST / GET / PUT atau PATCH / DELETE
Data encodingJSON atau XML sesuai API
Configuration managementAnsible collection cisco.ios
Infrastructure lifecycleTerraform provider
Controller integrationNorthbound API; southbound ke perangkat

Read-only/pre-check pada control node:

curl --cacert lab-ca.pem -H "Authorization: Bearer $TOKEN" <API_URL>
ansible-playbook -i inventory.yml baseline.yml --check --diff

Apply setelah diff direview dan perubahan disetujui:

ansible-playbook -i inventory.yml baseline.yml --diff

Perhatian: blok kedua benar-benar menerapkan perubahan. Token/password harus berada di environment atau vault, bukan source. Diff dapat mengandung data sensitif; lindungi output log. Gunakan pre-check, validation, approval, canary, post-check, dan rollback.

Cara memakai rangkuman
  1. Gunakan sebagai template, bukan konfigurasi universal.
  2. Ganti placeholder dan pisahkan blok multi-perangkat sebelum menempel.
  3. Verifikasi state dengan command show yang tersedia.
  4. Command pengayaan/platform-dependent harus diuji pada CML/GNS3/IOS yang sesuai.
